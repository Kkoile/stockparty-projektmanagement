1.                  Introduction

[The introduction of the Software Requirements Specification (SRS) should provide an overview of the entire document. It should include the purpose, scope, definitions, acronyms, abbreviations, references, and overview of the Software Requirements Specification.]

Das Projekt soll die technische Realisierung einer "Stock-Party" ermöglichen. Bei einer Stock-Party werden die Preise der Getränke ähnlich wie an der Börse durch die Nachfrage steigen oder fallen. Hierbei soll ein Kassensystem mittels RFID Karten realisiert werden. Es muss möglich sein Karten mit einem Geldbetrag aufzuladen und später an der Kasse die Getränke damit zu bezahlen. Für das Gesamte Projekt müssen eine Web-Apps für die Kasse, die Aufladen-Kasse, der Börsen-Monitor sowie eine Admin Konsole entwickelt werden. Außerdem soll ein Alkohol-Tester in das System eingebunden werden, welcher Einfluss auf dem Preis haben kann. Für die Anbindung des RFID-Readers muss ein node.JS Treiber entwickelt werden. Des Weiteren soll es möglich sein, mit einer paralel laufenden Stock-Party an einem anderen Ort konkurierend zu trinken. Hierfür soll auf beiden Partys ein Monitor eingerichtet werden, welcher einen Vergleich des Getränkekonsums darstellt.
